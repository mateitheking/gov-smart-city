# SHEET_SCHEMA — Smart Citizen MVP

This document defines the **single source of truth** schema for Google Sheets used by the Smart Citizen MVP.

**Hard rule:** column names and enum values must match exactly.  
If you change names/order, update bot + Apps Script accordingly.

---

## 1) Spreadsheet Setup

Create a Google Sheet with:
- Tab (worksheet) name: **`Requests`** (or match `REQUESTS_SHEET_NAME` in `.env`)
- Header row in **Row 1**
- One request per row (starting Row 2)

---

## 2) Columns (Requests tab)

> ✅ Recommended: keep this exact order for easiest implementation.

| Column | Type | Required | Example | Notes |
|---|---|---:|---|---|
| request_id | string | ✅ | `SC-1042` | Unique short ID, generated by backend/script |
| created_at | string (ISO) | ✅ | `2026-02-27T19:05:12+05:00` | Timestamp at creation |
| chat_id | number/string | ✅ | `123456789` | Telegram chat id used for notifications |
| telegram_user_id | number/string | ⛔ | `987654321` | Optional, if you store it |
| user_name | string | ⛔ | `Ami` | Optional profile field |
| phone | string | ⛔ | `+77001234567` | Optional profile field |
| description | string | ✅ | `Large pothole near school gate` | Main issue text |
| category | enum | ✅ | `Roads` | Can be `Unsorted` initially |
| priority | enum | ✅ | `High` | Based on AI/keywords/operator |
| confidence | number (0–1) | ⛔ | `0.74` | AI confidence; can be blank if AI off |
| tags | string | ⛔ | `pothole;asphalt;danger` | Optional; semicolon-separated |
| lat | number | ⛔ | `54.87012` | If user shared geo |
| lng | number | ⛔ | `69.15044` | If user shared geo |
| address_text | string | ⛔ | `Petropavl, Abay st, 10` | Manual address or уточнение |

## 3) Required Validation Rules (MVP)

### 3.1 Required fields to accept a request
A request is considered valid if:
- `description` is not empty AND
- at least one location is provided:
  - (`lat` and `lng`) OR `address_text`

### 3.2 Optional but recommended
- If `status` is `Need info` or `Rejected`, `public_comment` should be filled.

---

## 4) Enumerations (Must Match Exactly)

### 4.1 `status` values
| Value | Meaning |
|---|---|
| `New` | Received, not yet processed |
| `In progress` | Assigned / being handled |
| `Need info` | Citizen must clarify |
| `Done` | Resolved |
| `Rejected` | Invalid / spam / not under jurisdiction |

### 4.2 `category` values (MVP set)
| Value | Meaning |
|---|---|
| `Roads` | Potholes, asphalt, markings, sidewalks |
| `Lighting` | Street lights, lamps not working |
| `Trash` | Garbage, containers, illegal dumping |
| `Utilities` | Water/sewage/heating (optional) |
| `Safety` | Sparks, open manholes, hazards |
| `Unsorted` | AI low confidence / needs manual review |

> You can add more categories later, but keep `Unsorted`.

### 4.3 `priority` values
| Value | Meaning |
|---|---|
| `Low` | Non-urgent |
| `Medium` | Standard |
| `High` | Dangerous / urgent |

---

## 5) Notification Logic Contract (Apps Script)

The notification trigger **must** send a message when:
- `status` changes in a row AND
- `chat_id` exists

Recommended anti-duplicate mechanism:
- Compare new `status` with `last_notified_status`
- If different → notify → set `last_notified_status = status`

When notifying, include:
- `request_id`
- new `status`
- `public_comment` (if not empty)

---

## 6) Public Transparency (Optional, Safe Version)

If you need “public complaints view”, do **NOT** publish the Requests tab.

Create a second tab: **`Public_Requests`** with only safe fields:
- `request_id`, `created_at`, `category`, `priority`, `status`
- approximate area (no house number), e.g. `district` or `street_only`

Never publish:
- `chat_id`, `telegram_user_id`, `phone`, `user_name`, exact address, raw photos
