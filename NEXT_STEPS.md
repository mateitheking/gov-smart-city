# Что сделать дальше (практичный roadmap)

Ниже — приоритизированный список, чтобы довести MVP до рабочего пилота для акимата.

## P0 — Критично для запуска

1. **Реализовать backend-слой в Apps Script**
   - Сделать `doPost` (создание заявки) и `doGet` (получение списка заявок пользователя).
   - Добавить валидацию обязательных полей: `chat_id`, `description`, `location`.
   - Включить проверку `shared secret` и лимит запросов (anti-spam).

2. **Сделать триггер уведомлений о смене статуса**
   - `onEdit` в Google Sheets.
   - Отправка сообщения в Telegram при изменении `status`.
   - Защита от дублей (например, через `last_notified_status`).

3. **Закрыть security-базу**
   - Убедиться, что секреты только в `.env`/Script Properties.
   - Не логировать токены и приватные данные пользователей.
   - Пройтись по репозиторию и убрать возможные артефакты с данными.

## P1 — Стабильность и сопровождение

4. **Добавить базовые автопроверки**
   - `eslint` + единые правила форматирования.
   - Минимальные smoke-тесты для бота и web-формы.
   - CI pipeline (например, GitHub Actions): install + lint + test.

5. **Сделать персистентные сессии бота**
   - Сейчас in-memory сессии подходят только для демо.
   - Для прод/пилота вынести состояние в Redis/DB.

6. **Улучшить наблюдаемость и диагностику**
   - Структурированные логи (`request_id`, `chat_id`, `status`).
   - Единый формат ошибок между bot/web/apps-script.

## P2 — Продуктовые улучшения

7. **Доработать web-форму до parity с Telegram-ботом**
   - Полная отправка в backend, а не только демо-UX.
   - Единые валидации полей и одинаковые статусы/категории.

8. **Сформировать операторский регламент**
   - SLA по статусам (`New` → `In progress` и т.д.).
   - Шаблоны комментариев для граждан.
   - Роли и ответственность отделов.

9. **Подключить AI-классификацию безопасно**
   - Чёткий confidence-threshold.
   - Любая низкая уверенность → `Unsorted`.
   - Логи по качеству классификации и ручной корректировке.

## Definition of Done для MVP-пилота

- Заявка создаётся из Telegram и web-формы.
- Оператор меняет статус в Google Sheets.
- Пользователь получает уведомление в Telegram.
- В "Мои обращения" видно актуальный статус и комментарий.
- Есть базовые проверки качества (lint/smoke) и инструкция запуска для команды.
